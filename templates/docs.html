Technical Methodology

Harmonizer Pro utilizes a hybrid approach of traditional computer vision and neural semantic segmentation to achieve realistic style transfer across batch datasets.

01. MVC Architecture

The system is built on the **Model-View-Controller (MVC)** design pattern:

  • Model: SQLite database managing Project and ResultImage entities.
  • View: Responsive Jinja2 templates styled with Tailwind CSS.
  • Controller: Flask application handling routing and orchestration of the processing engine.

02. Smart Mode (U²-Net)

To prevent skin tone distortion and "filter burn," Smart Mode employs the **U²-Net** model for semantic subject segmentation.

// Logic Flow
1. Detect foreground mask via rembg (U²-Net)
2. Generate Gaussian-blurred alpha channel
3. Apply Histogram Matching to background only
4. Re-composite original subject onto harmonized background

03. Histogram Matching

The core stylization is achieved through Multi-channel Histogram Matching. The system extracts the cumulative distribution function (CDF) of the reference style and remaps the source pixel intensities to align with the target atmosphere.
